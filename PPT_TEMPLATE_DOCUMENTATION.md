# IEEE Presentation Template (Blue Version) — Detailed Documentation

## 1) Template Overview

This repository contains a single Microsoft PowerPoint template file:

- `IEEE-Presentation-Template-Blue-Version.pptx`

The template is a **16:9 corporate deck** with a unified IEEE visual language, one slide master, and 21 prebuilt example slides that demonstrate the available layouts and content patterns.

---

## 2) Important Usage and Brand Rules

The first slide in the deck includes mandatory usage guidance. Follow these rules when distributing or editing presentations made from this template:

1. Use this as a **Microsoft PowerPoint template**.
2. Do **not** modify the IEEE Master Brand or IEEE tagline elements.
3. Avoid round-tripping through non-PowerPoint formats if possible (for example, converting PowerPoint → Google Slides → PowerPoint), because this can introduce formatting/compatibility issues.
4. If you need an alternate format, use official versions from the IEEE Brand Experience site.
5. Questions can be submitted to `www.ieee.org/exd`.

- **File format:** `.pptx` (Open XML PowerPoint)

These settings mean the template is optimized for modern displays, virtual presentations, and conference screens.

## 6) Recommended Authoring Workflow

1. **Duplicate the template file** and rename it for your event/project.
2. Build your deck by selecting predefined layouts from **Home → New Slide**.
3. Replace placeholder text and images only; avoid rebuilding structural shapes.
4. Keep headings short and scannable; reserve dense details for speaker notes.
5. Use divider slides between major sections for narrative pacing.
6. Choose one of the 18pt/15pt/13pt body variants based on content density.
7. For image slides, prioritize high-resolution assets with consistent color tone.
8. Keep brand marks/tagline untouched per IEEE rules.

---

## 7) Content Design Guidance by Layout Type

### Title slides (Layouts 1–8)
Use when introducing the talk or a major transition with strong visual impact.

- Ideal for talk title, speaker name, affiliation, event name/date.
- Select the image composition variant that best fits your hero image.
- Avoid placing paragraph-length text on title slides.

### Divider slides (Layouts 9–13)
Use to separate agenda blocks.

- Keep text to a short section title (+ optional subtitle).
- Prefer high-contrast wording and avoid clutter.

### Body slides (Layouts 14–17)
Use for most explanatory content.

- **18pt:** low-density content (executive summaries).
- **15pt:** medium density (default for most slides).
- **13pt:** higher density; use sparingly and keep readability in mind.
- **Two-column:** compare alternatives, pros/cons, or parallel tracks.

### Image-assisted content (Layouts 18–19)
Use for case studies, diagrams, prototypes, and outcomes.

- Anchor each image with one takeaway sentence.
- Align captions and avoid mixing too many visual styles.

### Title-only / Blank (Layouts 20–21)
Use for custom endings, Q&A, or special-purpose visuals.

- Title-only works well for “Thank You”, “Q&A”, or “Backup Slides”.
- Blank is useful for full-bleed visuals or imported diagrams.

## 8) Accessibility and Readability Checklist

Before finalizing, verify:

- Adequate contrast between text and background.
- Minimum readable font size for projected environments.
- Consistent heading hierarchy and terminology.
- Alt text on informative images/charts where possible.
- No critical information conveyed by color alone.

---

## 9) Compatibility & Risk Notes

- Prefer editing in desktop PowerPoint to preserve exact layout behavior.
- Avoid repeated format conversions across tools.
- Embed or distribute linked assets carefully when sharing.
- Review all slides in presenter mode before delivery.

---

## 10) Quick Start (One-Minute Setup)

1. Open `IEEE-Presentation-Template-Blue-Version.pptx`.
2. Save as a new deck name.
3. Set your title slide (choose one of layouts 1–8).
4. Add section dividers (layouts 9–13).
5. Build core content with layouts 14–19.
6. End with layout 20 (or 21 for custom close).
7. Run slideshow and verify readability at full screen.
